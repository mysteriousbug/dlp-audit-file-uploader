Access Management - Comprehensive Walkthrough Questions
YOUR CORE QUESTIONS (Enhanced with Follow-ups)
1. Explain the access management process end to end
Main Question: Can you walk me through the complete access management lifecycle from initial request to access revocation?
Follow-up Questions:
What triggers an access request? (New hire, role change, project requirement?)
What system/tool is used to submit access requests? (ServiceNow, JIRA, email?)
What information is mandatory in an access request? (Business justification, requested privileges, duration, approver?)
Who are the approvers at each stage? (Manager → Data Owner → Security → DBA?)
How long does the typical provisioning process take? What's the SLA?
Is there a difference in process for standard access vs. privileged (DBA) access?
How do you handle emergency/urgent access requests?
What happens during role changes or transfers? Is there a review and adjustment process?
How is access revoked? (Termination, role change, project completion?)
What's the notification mechanism for terminations? (HR system integration, manual email?)
What's the SLA for access revocation upon termination?
Is there an audit trail for the entire lifecycle? Can you demonstrate it?
2. Can you show me how access is configured (in any one DB class among PostgreSQL, DB2, MongoDB, MSSQL, Oracle DB)?
Main Question: Can you provide a live demonstration of how access is configured in [select one: PostgreSQL/DB2/MongoDB/MSSQL/Oracle]?
Follow-up Questions:
Access Levels: What are the different access levels/roles available? (Read-only, Read-Write, Admin, DBA, etc.)
Role Assignment: How do you assign roles to users? (Database-level, schema-level, table-level?)
Least Privilege: How do you ensure least privilege principle is applied?
Access Matrix: Is there a documented access matrix/privilege matrix for this database? Can I see it?
Granular Permissions: Can you show an example of granular permissions? (e.g., SELECT on specific tables, EXECUTE on specific stored procedures)
Object-Level Security: How do you restrict access to specific schemas, tables, or sensitive columns?
Special Privileges: What about special privileges like CREATE, DROP, ALTER, GRANT? Who has these?
System/Admin Accounts: Are there any default or system accounts? How are they secured?
For Live Demo - Ask to See:
User creation command/process
Role assignment
Permission grants
Current access configuration for a sample user
How to revoke access
3. Can I get a list of all DB owned IDs?
Main Question: Can you provide a complete inventory of all database-owned/privileged IDs (DBA accounts, admin accounts, system accounts)?
Follow-up Questions:
Account Types: Can you categorize these by account type? (Individual DBA accounts, shared admin accounts, service accounts, emergency accounts?)
Naming Convention: Are DBA accounts individual (named after person) or generic (e.g., "dba_admin")?
Account Count: How many privileged accounts exist per database type?
Ownership: For each privileged account, who is the owner/responsible person?
Last Used: Can you provide the last login/usage date for each account?
Active vs Inactive: Are any of these accounts inactive or dormant?
Service Accounts: Which of these are service/application accounts vs. human users?
Break-Glass Accounts: Do you have emergency "break-glass" accounts? How many? How are they secured?
Specific Data Request:
Please provide a report with the following columns:
Database Name/Instance
Database Type (Oracle, Postgres, etc.)
Account Name/Username
Account Type (Individual DBA, Shared Admin, Service Account, etc.)
Owner/Responsible Person
Last Login Date
Account Status (Active/Inactive/Disabled)
Creation Date
Special Privileges (SUPERUSER, SYSADM, sa, root, etc.)

5. Can you tell me more about unauthorized privilege escalation? How do you identify unauthorized changes?
Main Question: How do you detect and prevent unauthorized privilege escalation in your databases?
Follow-up Questions:
Prevention Controls:
What controls prevent users from escalating their own privileges?
Can regular users grant privileges to themselves or others? How is this restricted?
Are administrative commands (GRANT, ALTER USER, CREATE USER) restricted to authorized DBAs only?
Do you have segregation of duties between DBAs who can grant access vs. those who manage infrastructure?
Are there any privileged commands that require dual authorization (maker-checker)?
Detection & Monitoring:
What monitoring is in place to detect privilege escalation attempts?
Are all privilege grants/modifications logged? Where are these logs stored?
What specific events trigger alerts? (Examples: GRANT SYSADM, ALTER USER, adding users to admin groups)
How quickly are suspicious privilege changes detected?
Can you show me examples of configured alerts for privilege escalation?
Change Validation:
How do you baseline authorized privileges for each user?
Is there a process to regularly compare current privileges against approved access matrix?
How frequently do you validate that privileges haven't changed without authorization?
What tools do you use for privilege drift detection?
Incident Response:
What happens when unauthorized privilege escalation is detected?
Can you provide examples of past incidents and how they were handled?
What is the escalation path for security incidents related to access?
6. Are there access controls to restrict access to sensitive information? What tool are you using to monitor database activity?
Main Question: How do you protect sensitive data within databases, and what monitoring tools are in place?
Follow-up Questions:

Schema-level restrictions?
Table-level restrictions?
Column-level security/masking?
Row-level security?
Do you use data masking or dynamic data masking for non-production environments?
Are sensitive columns encrypted? What encryption is used?
Who has access to sensitive data, and how is this justified?
How do you enforce "need-to-know" principle for sensitive information?
Database Activity Monitoring (DAM):
What tools are you using for database activity monitoring?
Native database auditing? (Oracle Audit Vault, pg_audit, SQL Server Audit, etc.)
Third-party DAM solution? (Imperva, IBM Guardium, McAfee, etc.)
SIEM integration? (Splunk, QRadar, ArcSight, etc.)
Monitoring Scope:
What activities are monitored and logged?
All queries? Or only specific types (DDL, DML, DCL)?
Failed login attempts?
Privileged user activity?
Access to sensitive tables/schemas?
Data exports/downloads?
Configuration changes?
Are queries logged with actual SQL statements or just metadata?
Is there any performance impact from monitoring? How is this managed?
Alerting:
What real-time alerts are configured?
Failed authentication attempts (brute force detection)?
Unusual query patterns?
Access to sensitive data outside business hours?
Bulk data exports?
Privilege escalation attempts?
Execution of dangerous commands (DROP, TRUNCATE)?
Who receives these alerts? What's the response process?
Can you show me examples of recent alerts and how they were resolved?
Log Management:
Where are database audit logs stored?
How long are logs retained? (Regulatory requirement?)
Are logs encrypted and protected from tampering?
Who has access to audit logs?
Are logs backed up? What's the backup retention?
Can you demonstrate log retrieval for a specific user activity?
Reporting:
What standard reports are generated from DAM tools?
Can you show me a sample activity report for a privileged user?
Do you have reports on sensitive data access?
Are reports provided to management or compliance teams?
